# Tracker

The live table. Milestones run top to bottom **in sequence**; each delivers a tangible result (~4–6 weeks). Each item ≈ one week of work with a weekly **Due Date**. An item closes when its validation is in `validation/`; a milestone closes when all its items do. A moved **Due Date** is a **logged slip** — never overwrite it. 2+ slips on an item → flag.

- **status:** `open` · `in-progress` · `blocked` · `closed`
- **slips:** count + moves, e.g. `1 (6/25→7/2)`
- **validation:** path into `validation/`, e.g. `validation/2026-06-25-vendor-msa.pdf`
