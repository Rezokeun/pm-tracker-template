# Session Log

Append-only, **newest on top**. Written as a side effect of changes (see `CLAUDE.md` → Logging), so it builds itself — nothing to remember at session end. Every line is dated and traces to a tracker row, a validation file, `source-log.md`, or `notes.md`. This is the digestible **digest** of what changed: read the top to see what's happened recently. (The full verbatim input behind it lives in `source-log.md`.)

Tags: `[in]` inbound · `[done]` item closed · `[slip]` date moved · `[note]` reference added · `[decision]` · `[flag]` 2+ slips.
