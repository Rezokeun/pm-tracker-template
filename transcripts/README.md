# transcripts/

Per-session verbatim transcripts — one file per working session, named `YYYY-MM-DD-session.md` (add `-2`, `-3` for additional sessions the same day). Each captures the **full exchange** — the owner's input *and* the agent's responses — written as the session proceeds (write-ahead), so nothing is lost if a session ends.

This is provenance: the truth-anchor behind summaries. Read it by date or search when you need to verify or re-derive something the tracker, `session-log.md`, or `notes.md` distilled. It is **not** a browsable layer (that's `notes.md`) or an event digest (that's `session-log.md`).

Note on fidelity: the agent captures your input exactly; it records its own responses faithfully but may not be byte-perfect. Claude Code also keeps a literal session log under the hood as a forensic backstop if you ever need it.
